# BFD：

​ 双向转发检测，是一个介质无关，协议无关的故障检测协议，应用广泛，能够再毫秒级别感应到网络故障问题

注：BFD 通过在设备之间建立会话，互相交互 BFD 报文来检测两者之间的连通性问题

# BFD 会话建立方式：

​ 静态建立 BFD 会话：管理员手工配置本地标识符和远端标识符（标识符用于区分本地会话）

​ 动态建立 BFD 会话：标识符由系统自动分配

# BFD 的状态变化：

​ Down：建立 BFD 会话的初始状态 / 会话 Down

​ init：处于 Down 状态时收到了 Down 状态 BFD 报文进入 init 状态

​ UP：处于 init 状态时收到了 init/UP 状态的报文进入 UP 状态，表示 BFD 会话建立完成

​ AdminDown：管理员 Down，由管理员手工将会话 Down，不是因为网络故障

注：在任何一个状态，只要会话超时就会回到 Down 状态，此时认为网络故障

# BFD 的检测模式：

​ 异步模式：两个设备都支持 BFD，互相交互 BFD 报文，设备通过是否能够接收到对端发送的 BFD 报文来判断会话连接状态（连通性）

​ 查询模式：只有一端开启 / 支持 BFD，通过是否能够接收到自己发送出去的 BFD 报文判断连通性

# BFD 检测时间：

​ TX：发送时间间隔

​ RX：接收时间间隔

​ DM：检测倍数

​ 设备实际发送时间间隔 = MAX (本地配置发送时间间隔，对端配置接收时间间隔)

​ 设备实际接收时间间隔 = MAX (对端配置发送时间间隔，本地配置接收时间间隔)

​

# 实际检测时间：

​ 异步模式 = 设备实际接收时间间隔 * 对端配置检测倍数

​ 查询模式 = 设备实际接收时间间隔 * 本地配置检测倍数

静态 BFD

1. bfd //开启BFD
2. bfd 1 bind peer-ip *  source-ip * //创建BFD会话
3. discriminator  local  *  //设置本地标识符
4. discriminator  remote  * //设置远端标识符
5. commit  //启动配置

动态 BFD

  1. bfd // 开启 BFD
  2. bfd 1 bind peer-ip * source-ip * auto // 创建 BFD 会话
# BFD Echo：

BFD 单臂回声，当进行 BDF 检测时，有一端不支持 BFD 功能，此时可以通过回声达到检测的目的（设备发送 BFD 报文封装源目 IP 地址都为自己的地址，对端接收到后根据目的 IP 转发回来）

配置:

​ bfd 1 bind peer-ip 10.1.12.2 interface g0/0/0 one-arm-echo

​ discriminator local 100

​ commit

# 静态路由与 BFD 联动：

​

1. 建立静态路由的主备份路径：

（r1）ip route-static 10.1.45.0 255.255.255.0 10.1.34.4
（r3）ip route-static 10.1.45.0 255.255.255.0 10.1.35.5 preference 70

2. 在静态主路径上与 BFD 进行联动

（r1）bfd 1 bind peer-ip 10.1.34.4 source-ip 10.1.34.3 auto

（r2）bfd 1 bind peer-ip 10.1.34.3 source-ip 10.1.34.4 auto

3. 将静态主路径上与 BFD 进行联动

（r1）ip route-static 10.1.45.0 255.255.255.0 10.1.34.4 track bfd-session 1

修改 BFD 时间间隔：

  1. detect-multiplier 5 // 修改检测倍数
  2. min-tx-interval 500 // 修改发送报文时间
  3. min-rx-interval 500 // 修改接收报文时间
# OSPF 与 BFD 联动

在 OSPF 进程下：

bfd all-interfaces enable // 开启所有 OSPF 接口的 BFD 检测功能

bfd all-interfaces min-tx-interval

bfd all-interfaces min-rx-interval

bfd all-interfaces detect-multiplier
