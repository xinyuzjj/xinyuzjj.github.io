# 交换机:

​ 工作在数据链路层

​ 工作原理：根据数据帧中的目的 MAC 地址进行转发 查表转发

​ 每一个接口属于不同的冲突域，交换机可以隔绝冲突域

​ 交换机的所有接口属于同一个广播域

​ 广播域：一个广播报文所能达到的所有范围

# 交换机的 MAC 地址表项学习：

​ 交换机接受到一个数据帧，查看源 MAC 地址，将源 MAC 地址于接收到数据帧的接口作为表项加入到 MAC 地址表中

​ MAC 地址表项的老化时间 300s

# 交换机的 MAC 地址表刷新：

​ 当交换机收到一个数据帧，查看源 MAC 地址，是否存在 MAC 地址表中，不存在则学习，如果存在则将 MAC 地址的老化时间刷新。

# 交换机对数据帧的转发方式：

​ 转发：交换机从一个接口收到数据帧，只从另外一个接口发出

​ 当收到一个已知的单播数据帧时，交换机会进行转发操作，已知的单播数据帧（数据帧的目的 MAC 地址在 MAC 地址表中存在）

​ 泛洪：交换机从一个接口收到一个数据帧，从出这个接口以外的所有接口进行发出

​ 当收到一个未知的单播数据帧，组播帧，广播帧，未知单播数据帧（目的 MAC 地址在 MAC 地址表中不存在）

​ 丢弃：交换机从一个接口收到数据帧后，不会从任何接口进行发出

​ 当交换机收到错误的数据帧（尾校验不通过）、数据帧的入接口和出接口一致

# 网络层：

​ 在上层数据前加上 IP 头，数据包

​ 网络层协议：IP ,ICMP ,IPX 等

​ 路由器工作在网络层

​ 路由器的每个接口属于不同的广播域

​ 网关：收集本网段信息，发送给其他网段。

# IP 头部参数：

​ 版本（Version）：表识使用 IP 的版本，ipv4 的值为 4

​ 头部长度（header length）：用于表识 ip 头部长度

​ 服务类型（Type of Service）：主要用于 QOS，给不同的流量进行限速

​ 总长度（Total Length）：表识整个 ip 报文的总长度

​ 标识位（Identification）：用于报文重组

​ 标志位（Flags）：用于标识报文能不能分片，是否时最后一个包分片

​ 保留位：为 0，没由意义

​ 分片位：为 1 时表示不允许分片，为 0 时表示允许分片

​ 更多位：如果为 0 时，表示时最后一个包分片，设备收到后可以进行重组报文的操作

​ 片偏移（Fragment Offset）：表示分片在原报文的偏移位置，用于重组报文时能够正确排列位置

​ 生存时间（TTL）：用于三层（网络层防环），初始为 255，每经过一台三层设备减一，当路由器收到一个数据包生存时间为 0 的，直接将数据包进行丢弃

​ 协议（Protocol）：用于标识上层协议类型

​ 1：ICMP

​ 2：IGMP

​ 6：TCP

​ 17：UDP

​ 头部校验和（Header Checksum）：用于校验 IP 头部完整性

​ 源 IP（Source IP Address）：发送数据包的设备的 IP 地址

​ 目的 IP（Destination IP Address）：最终接收者的 IP 地址

​ 可选项（Options）：用于携带一些可选能力，例如认证等

​ 填充（Padding）：无意义，维持 IP 头部为 4 字节的整数倍

IP 头部长度 为 20–60 字节，当不存在可选项时为 20 字节

# IP 地址：

​ 长度 32bit 位，采用点分十进制表示

​ 范围 0.0.0.0-----255.255.255.255

​ 组成：网络 + 主机

​ 网络：用来标识一个网络

​ 主机：用来标识网络中的一台主机

​ 掩码：用于标识一个 IP 地址中的网络部分以及主机部分，掩码为连续的 1 和连续的 0 组成。当该位置为网络位置时，掩码位为 1，当该位置为主机位时，掩码位为 0。
