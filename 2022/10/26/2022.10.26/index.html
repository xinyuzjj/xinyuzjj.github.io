# OSPF 的工作原理 / 工作过程:

​ 1. 运行 OSPF 协议的路由器互相发送 hello 报文用于发现，建立，维护邻居关系

​ a）DOWN：OSPF 的初始化状态，此状态下不会发送 hello 报文等

​ b）Attempt 状态：发送 hello 报文但是未接收到邻居发送的 hello， 此状态在广播网络中不存在，只存在于特殊网络类型

​ c）init：发送了 hello 报文，也收到了 hello 报文，但是，收到的 hello 报文的邻居列表中没有自己的 router id

​ d) 2-way: 发送了 hello 报文，也收到了 hello 报文，在收到的 hello 报文中邻居列表有自己的 router id，此状态为邻居最高状态，标识已经建立完成邻居关系

​ 2. 协商主从，交互 LSA，同步 LSDB

​ a) EXstart：互相发送空的 DD 报文，携带 router id 用于协商主从，router id 大的作为主设备，后续主设备发送任何消息从设备都需要进行确认。

​ b）Exchange：互相发送 DD 报文，用于描述本地 LSDB 得摘要信息。

​ c）loading：使用 LSR，LSU，LSACK 进行同步 LSDB，如果设备不需要向对方请求 LSA 可以直接跳过此状态

​ d）Full：已经和邻居同步完成 LSDB，进入此状态，此状态为邻接关系最高状态

​ 3. 基于 LSDB，使用 SPF 算法，以自己为根构建最短路径树，将去往每一个目的网段的最优路由进行加入到路由表

#
# OSPF 支持的网络类型：

​ 广播：以太网环境默认网络类型就是广播网络，此网络类型设备能够与此网络中的所有设备建立邻居关系

​ NBMA (非广播多路访问网络)：需要特定的网络设备支持（FR：帧中继），此网络类型下设备能与网络中的所有设备建立邻居关系，但是并非是广播型网络

​ P2P（点到点）：此网络下的设备接口对端只存在一台设备，和该设备建立邻居不需要协商掩码

​ P2MP（点到多点）：此网络类型默认不存在，只能通过其他网络类型进行修改，设备只能和该网络中的部分设备建立邻居关系。

# OSPF 的路由器角色：

​ DR (指定路由器)：和所有运行了 OSPF 的路由器建立邻接关系

​ BDR (备份指定路由器)：和所有运行了 OSPF 的路由器建立邻接关系

​ DRother (除 DR 和 BDR 以外的其他所有设备)：统称为 DRother，只会和 DR 和 BDR 之间建立邻接关系。

# DR 和 BDR 的作用：

  1. 减少邻接关系数量

  2. 减少相同的 LSA 泛洪

  3. 减少链路资源浪费

  4. 减少设备资源损耗

    DR 与 BDR 的选举规则：

    1. 比较优先级，以大为优 0-255

      ​ a）优先级为 0 的路由器不参与选举

      2. 当优先级一致时，比较 router id

      ​ a）以大为优

      # DR 和 BDR 的选举：

    2. 原来网络中存在 DR 和 BDR 设备：

      a） 新接入设备在收到网络中的 hello 报文后，查看发现 hello 报文中存在 DR 和 BDR 设备

      b） 无论设备的优先级是多少，都会将自己作为 DRother，和 DR,BDR 去建立邻接关系

    3. 原来网络中不存在 DR 和 BDR 设备

      a）当设备接入网络时，监听网络中的 hello 报文，发现网络中不存在 DR 和 BDR 设备

      b）发送自己的 hello 报文进行选举

      c）网络中先选举 BDR 设备比较优先级和 router id

      d）将选举出来的 BDR 升级成为 DR 设备

      e）再从所有的 DRother 中选举一个 BDR 设备

    DR 和 BDR 选举根据网段进行选举。

    # 区域：

    ​ 使用 area id 去标识

    ​ 骨干区域：area 0 区域，负责所有跨区域的流量转发

    ​ 非骨干区域：非 area 0 区域，所有的非骨干区域都必须和骨干区域相连

    # OSPF 的防环机制：

    ​ 域内防环：使用 SPF 算法构建最短路径树，天然无环

    ​ 域间防环：所有的非骨干区域必须和骨干区域相连，非骨干区域之间通信必须经过骨干区域

排除命令：[Huwei] dis ospf error
