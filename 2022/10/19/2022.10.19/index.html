# 有类 IP 地址：

A 类：0.0.0.0-127.255.255.255 /8 0

B 类：128.0.0.0-191.255.255.255 /16 10

C 类：192.0.0.0-223.255.255.255 /24 110

D 类：224.0.0.0-239.255.255.255 用于组播 1110

E 类：240.0.0.0-255.255.255.255 保留 1111

# 网络地址：

用于标识一个网络，主机位全为 0

例如：192.168.1.0/24

# 广播地址：

用于代表网络中的所有主机，主机位全为 1，朝广播地址发送数据包时，所有的主机都要接收并处理

例如：192.168.1.255/24

可用地址数：

一个网段中可以分配主机使用的 IP 地址

​ 数量 = 2n-2 //n 等于主机位数

# 公网地址：

由 IANA 组织，分配给运营商使用，企业一般去运营商租用公网 IP 地址。

# 私网地址：

在企业中某些设备不需要连接 Internet，可以使用私网地址，只要在一个网络中不重复即可，在 A，B，C 类中都有一段私网地址

A 类：10.0.0.0–10.255.255.255

B 类：172.16.0.0–172.31.255.255

C 类：192.168.0.0–192.168.255.255

# 特殊地址：
# 有限广播

255.255.255.255 作为目的时，同一网络中的所有主机都能收到该报文，并需要进行处理

# 任意地址

0.0.0.0 可以用此地址代表所有的 IP 地址

# 环回地址

127.0.0.0 -127.255.255.255 用于测试自身的状态

# 本路链路地址

169.254.0.0/24 用于当自动获取地址失败后，临时链路通信

# VLSM:

可变长子网掩码

​ 原理：网络位向主机位借位

​ 作用：减少广播域，减少同一网络中的主机数量

​ 划分方式：

​ 1. 按照主机数量进行划分

​ a) 按照从大到小的主机数量进行计算

​ 2. 按照子网数量进行划分

# ICMP：

互联网控制消息协议，工作在网络层，协议号为 1

# 作用：

用于传递网络设备之间的差错和控制信息，收集网络信息，排错和诊断。

# 主要的应用：

​ ping 和 tracert

​ Ping 作用：检测网络连通性

​ Tracert 作用：用于回显路由

# 报文格式：

Type + code +checksum + 报文内容

Type：消息类型

code：具体参数

校验和：用于检验数据完整性

报文内容：包含具体的信息

Type Code 描述
0 0 Echo Reply
3 0 网络不可达
3 1 主机不可达
3 2 协议不可达
3 3 端口不可达
5 0 重定向
8 0 Echo Request
# ICMP 重定向：

1. 主机访问服务器，两台设备处于不同的网段

2. 主机默认将所有跨网络访问的数据报交由自己的网关进行处理

3. 网关收到主机发送的报文后查找路由表，发现下一跳的地址与主机所处同一网段

4. 网关会将下一跳的 IP 地址通过 ICMP 重定向消息发送给主机，主机收到后。再次访问服务器时，直接通过网关给的下一跳进行访问。
