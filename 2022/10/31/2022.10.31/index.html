# VLAN 间通信：

​ 背景：划分完 VLAN 后，部分不同的 VLAN 之间有互访的需求，使用 hybrid 能够解决互访需求，但是，如果不同的 VLAN 使用不同的网段，hybrid 无法再满足需求。

​ 解决方法：

1. 部署路由器
2. 三层交换

1. 部署路由器：可以使用路由器的物理接口实现不同 VLAN 间的正常通信

​ a) 使用物理接口：每一个 VLAND 都需要对应一个物理接口，浪费接口

​ b）使用子接口（单臂路由）：使用一个路由器的物理接口逻辑虚拟成多个子接口，每个子接口对应一个 VLAN, 实现不同 VLAN 之间通信。

2. 三层交换：在交换机上创建三层接口用于作为 VLAN 下主机的网关，给交换机开通路由功能，能够进行查表转发的动作

# STP：

​ 生成树协议

​ 背景：在传统的二层网络中，容易出现单点故障，导致网络中的部分设备无法正常通信，此时如果部署冗余链路会导致出现广播风暴以及 MAC 地址漂移

# 三层网络环路解决：

​ 通过 IP 头怒的 TTL 字段进行防环，当 TTL=0 时，不再进行转发操作。

# 二次网络环路解决：

​ 无法通过帧头中的某个字段进行防环，只能引入其他协议帮助

# 名称解释：

​ port ID (端口 ID)：由端口优先级和端口号组成，用于确定端口角色

​ 端口优先级：默认 128

​ 取值范围：0-255

​ 步长：16

​ BID（桥 ID）: 由桥优先级和桥 MAC 两部分组成，用于选举根桥，优先比较优先级，相同时，比较 MAC 地址

​ 桥优先级：默认 32768

​ 取值范围：0-65535

​ 步长（取 0 或者 4096 的整数倍）：4096

​ Root (根桥)：在所有的交换机中选举出一台交换机作为根桥，基于根桥构建 STP 树

​ cost（开销）：链路或者接口的开销，不同的参考标准会使得相同的链路出现不同的 cost 值，建议将网络中的所有交换机的参考标准改为一致

​ RPC（根路径开销）：设备或者设备接口去往根桥的所有路径开销总和（入向接口的开销总和）

​ BPDU (网桥协议数据单元)：网络中的交换机通过交互 BPDU 来完成网络拓扑收敛，

​ 配置 BPDU:

​ 用于平时交互使用

​ TCN BPDU：

​ 用于当拓扑发生改变后发送

​ BPDU 参数：

​ PID: 协议 ID

​ PVI: 版本 ID

​ BPDUtype：BPDU 类型，用于标识该 BPDU 是配置 BPDU 还是 TCN BPDU

​ Flags：标志位，共 8bit，只使用了最高 bit 位和最低 bit 位，最高位是表示位 TCA, 最低置位表示位 TC

​ Root ID: 网络中根桥的桥 ID，初始状态下，网络中所有设备都认为自己是根桥

​ RPC: 根路径开销

​ BID：产生设备或者发送设备的桥 ID

​ Port ID: 产生设备或者发送设备的端口 ID

​ Massage Age：消息寿命，从根桥发送后，每经过一台网桥都会进行 + 1 操作，作用类似 TTL 值

​ Max age：最大生存时间，默认 20s

​ Hello time ：发送 BPDU 的时间间隔，默认 2s

​ Forward delay: 转发延迟时间，15s
