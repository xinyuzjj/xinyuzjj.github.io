# ARP:

​ 地址解析协议

​ 作用：通过 ip 地址去解析对端 MAC 地址，完成数据帧的封装，ip 地址通过 DNS（域名解析服务）得到

​ 分类：普通 ARP, 免费 ARP, 代理 ARP, 逆向 ARP

# 普通 ARP 过程：

​ PC1<---------------->PC2

​ 1.PC1 需要和 PC2 进行通信，ip 地址通过 DNS 解析得到，完成网络层封装，需要进行数据链路层封装，没有对端 MAC 地址

​ 2.PC1 发送 ARP 请求报文，源 ip 地址为 PC1，源 MAC 地址为 PC1，目的 ip 地址为 PC2，目的 MAC 地址为广播（保证对端能够收到并处理）

​ 3.PC2 收到 PC1 发送的请求报文，查看目的 MAC 地址为广播，需要自己处理，解封装网络层，为 ARP 协议，目的 ip 地址为自己的 ip 地址，需要自己处理

​ 4.PC2 朝 PC1 回应 ARP 响应报文，将 PC1 的 IP 地址 MAC 地址封装为数据的目的，将自身的 ip 地址，MAC 地址封装为源地址。

​ 5.PC1 收到 PC2 的响应报文后，将 PC2 的 ip 地址与 MAC 地址形成对应表项加入 ARP 缓存表中，后续通信过程直接将 PC2 的 IP 地址 MAC 地址封装为目的地址

# 代理 ARP：

​ PC1<------------>(1)AR1(2)<--------------------->PC2

​ 场景说明：PC1 与 PC2 属于同一网段，AR1 的 1 口和 2 口属于不同网段，（实现使用掩码不同即可）

​ 过程：

​ 接口内 arp-proxy enable // 在接口视图下开启代理 ARP 功能

​ 1.PC1 需要和 PC2 进行通信，需要获取 PC2 的 MAC 地址，由于 PC1 与 PC2 处于同一网段，会直接向 ip 为 PC2 的地址发送 ARP 请求，用于获取 PC2 的 MAC 地址

​ 2. 数据帧被路由器所接收，查看目的 MAC 地址，为广播处理，查看 IP 地址为 PC2 的地址与接口属于不同网段，此时接口开启代理 ARP，路由器会将 1 口的 MAC 地址发送给 PC1，欺骗 PC1 自己的接口 MAC 地址就是 PC2 的 MAC 地址，

​ 3.PC 收到后认为路由器 1 口的 MAC 地址为 PC2 的，后续封装数据帧时将路由器 1 口的 MAC 地址封装为目的 MAC 地址，

​ 4. 数据转发过程，路由器 1 口收到目的 MAC 地址为自己的 MAC 时继续解封装，目的 IP 不是自己的时，根据 ARP 表项进行转发操作

# 逆向 ARP (RARP)：

​ 使用目的 MAC 地址请求目的 IP 地址

# 免费 ARP:

​ 用于检测地址是否冲突

​ 原理：去请求自身 ip 地址的 MAC 地址，若收到回应，则认为 IP 地址冲突。

# IP 路由：

​ 不同网段需要进行通信，在二层网络中无法满足，需要进行跨二层网络进行转发数据。

# 路由名词和动词：

​ 名词：路由表中的一条路由信息

​ 动词：报文达到网络设备后需要转发进行查表得动作。

# 路由表六要素：

​ 目的网络 + 掩码：用来标识目的网段信息

​ 协议：用来标识这条路由通过何种方式得来

​ 直连：当路由器接口配置好 ip 和接口状态为 up 时，直接生成

​ 静态：通过管理员进行手工配置得来

​ 动态：通过路由协议进行学习路由

​ 优先级：标识路由的优先级，取值 0-255，以小为优

​ 直连：优先级为 0，最优，并且不允许修改

​ 静态：默认优先级 60 ，可以进行修改

​ 动态：优先级根据不同的路由协议有不同的值

​ 开销：去往目的网络的开销总和

​ 出接口：去往目的网段的出接口

​ 下一跳：出接口对端设备的接口 ip 地址

# 路由表加表原则：

​ 当收到两条去往同一目的网段的路由时，先比较优先级，优先级以小为优，将优先级小的加入路由表，如果优先级相同，比较开销，优先选择开销小的加入路由表。

# 路由最长匹配原则：

​ 当收到一个数据包时，将数据包中的目的 IP 地址与路由表中的路由进行匹配，再从匹配中的路由中选择最大掩码的路由对数据包进行指导转发操作。

# 路由器转发数据流程：

​ 路径上的每一台路由器进行逐条查表转发操作，意味者路径上的每一台路由器都需要有目的网段的路由信息。

# 静态路由：

​ 通过管理员手工配置得到

​ 优点：配置较为简单，适用于小型且稳定的企业

​ 缺点：无法自适应网络拓扑变化，维护麻烦

​ 配置命令：ip route-static 10.1.1.0 24 10.1.1.2 // 目的网段 + 掩码或掩码长度 + 匹配下一跳将数据包进行转发出去

# 缺省路由：

​ 是一种特殊路由，网段为全 0 掩码长度也为 0，能够匹配所有主机 ip 地址。

​ ip route-static 0.0.0.0 0 10.1.1.2
