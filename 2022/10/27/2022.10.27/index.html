# DHCP：

​ 动态主机配置协议

​ 背景：网络当中存在大量的主机，静态配置 ip 地址等网络参数时，工作量大，并且容易出现 ip 地址冲突等网络错误，或者容易出现用户自己修改网络参数导致无法上网。

​ 作用：为网络中的主机分配 IP 地址等参数，减少网络工程师的工作量，以及降低网络故障率。

# 报文：

​ DHCP discover（发现报文）：用于发现网络中的 DHCP 服务器。

​ DHCP offer（提供报文）：用于为主机提供 ip 地址等网络参数

​ DHCP request（请求报文）：

​ 1. 用于向服务器确认是否是自己的分配的参数

​ 2. 告知网络中的其他 DHCP 服务器回收为自己分配的 ip 等参数

​ DHCP ACK (确认): 对 request 报文的确认

​ DHCP NAK（否认）: 对 request 报文的否认

​ DHCP RELEASE（释放）: 主机主动释放 ip 地址等

# 地址池：

​ 用于承载为主机分配的 IP 地址等

​ 分类：接口地址池和全局地址池

​ 接口地址池：ip 地址范围，于接口 ip 地址同一段的地址都会分配给主机使用

​ 全局地址池：需要手工写地址范围

# 工作过程：

​ 1. 主机以广播形式发送 DHCP discover，用于寻找网络中的 DHCP 服务器

​ 2.DHCP 服务器以单播发送 DHCP office 报文，为主机提供 IP 地址等参数

​ 3. 主机以广播形式发送 DHCP request 报文，向服务器进行确认是否为自己分配的 IP 等参数，同时告知网络中其他 DHCP 服务器 自己使用的 ip，让他们进行回收为自己分配的参数

​ 4. 服务器以单播发送 DHCP ACK 同意主机使用自己分配的 IP 等参数

​

# DHCP 续租过程：

​ 1. 当主机的租期只剩 50% 时，会向原来的 DHCP 服务器以单播形式发送 DHCP request 报文，用于刷新租期

​ 2. 如果服务器愿意续租，则回应 ACK 报文，将主机的 IP 地址租期重置。如果不愿意则回应 NAK，不进行刷新租期

​ 3. 如果服务器由于网络或者自身原因没有收到，主机的 request，主机将继续使用原有的租期

​ 4. 当主机的租期只剩 12.5%，会以广播的形式发送 request 报文，用于寻址网络中的所有 DHCP 服务器，用于重绑定。

​ 5. 如果原来服务器回应 ACK，直接将地址刷新，如果回应 NAK，被动释放地址。或者等待租期结束后被动释放地址

​ 6. 后续会周期发送 discover 报文寻找网络中的 DHCP 服务。
