# BGP：

# AS:

自治系统，AS 和 AS 之间通过 AS 号进行区分，AS 号范围 1-65535，其中 1-64511 为公有 AS 号，64512-65535 为私有 AS 号

# BGP:

边界网关协议，用于 AS 与 AS 之间传递和控制路由，属于路径矢量路由协议

BGP 是应用层协议，基于 TCP 的 179 端口建立连接，只有建立了 TCP 连接，才能建立 BGP 连接，由于 TCP 为单播建立连接，所以 BGP 需要手工指定邻居，又因为 TCP 连接建立的基础是网络可达，所以 BGP 只要网络可达就可以建立邻居关系

# BGP 的邻居类型：

BGP 邻居关系称之为对等体关系

​ IBGP 对等体：处于相同 AS 的两个设备之间建立的 BGP 对等体关系，一般采用环回口建立（需要指定更新源地址为环回口）

​ peer 对端 IP 地址 as-number 对端 AS 号 // 接口地址建立

​ Peer 对端 ip 地址 Connect-interface loopback0// 环回口建立必须修改

​ EBGP 对等体：处于不同 AS 的两个设备之间建立的 BGP 对等体关系，一般采用直连接口建立，如果用环回口建立需要指定 ebgp 多跳（EBGP 对等体建立时默认 TTL=1）

​ peer 对端 IP 地址 as-number 对端 AS 号

​ Peer 对端 ip 地址 Connect-interface loopback0// 环回口建立必须修

​ peer 对等体地址 ebgp-max-hop 1-255（不设具体数据直接回车为最大值）

# BGP 的报文类型：

​ OPEN 报文：协商参数，建立对等体关系（版本，AS 号，BGP ID ,Hold Time）

​ UPdate 报文：传递路由信息（发布和撤销路由）

​ Notification：差错报文，当邻居关系发送错误，通过该报文通知对等体断开邻居关系

​ Keepalive：保活报文，周期性 60s 发送，用来维护对等体关系，

​ Route-refrash：路由刷新报文，当配置了路由策略，路由信息发生变化，发送该报文通知对等体

报文名称 作用 发送时刻
Open 协商 BGP 对等体参数，建立对等体关系 BGP TCP 连接建立成功之后
Update 发送 BGP 路由更新 BGP 对等体关系建立之后有路由需要发送或路由变化时向对等体发送 Update 报文
Notification 报告错误信息，中止对等体关系 当 BGP 在运行中发现错误时，发送 Notification 报文将错误通告给 BGP 对等体
Keepalive 标志对等体建立，维持 BGP 对等体关系 BGP 路由器收到对端发送的 Keepalive 报文，将对等体状态置为已建立，同时后续定期发送 keepalive 报文用于保持连接
Route-refresh 用于在改变路由策略后请求对等体重新发送路由信息。只有支持路由刷新能力的 BGP 设备会发送和响应此报文 当路由策略发生变化时，触发请求对等体重新通告路由
# BGP 的状态机：

​ Idle：初始状态，等待 Start 事件（peer…）

​ Connect: 尝试建立 TCP 连接，并启动一个计时器 32s，如果建立成功，则进入 Opensent 状态，建立失败进入 Active 状态，超时重启计时器停留在 connect 状态

​ Active：重新启动计时器，建立 TCP 连接，成功进入 Opensent，超时回到 connect，失败停留在 Active

​ Opensent：TCP 连接建立成功，开始发送 Open 报文，协商参数

​ Openconfirm：收到了 OPEN 报文并且协商成功进入该状态，开始发送 Keepalive 报文

​ Established：收到了 Keepalive 报文，对等体关系建立成功

注：在此期间只要收到了 Notification 报文都会回到 Idle 状态

看看：：：：

・Idle 状态是 BGP 初始状态。在 Idle 状态下，BGP 拒绝对等体发送的连接请求。只有在收到本设备的 Start 事件后，BGP 才开始尝试和其它 BGP 对等体进行 TCP 连接，并转至 Connect 状态。

▫Start 事件是由一个操作者配置一个 BGP 过程，或者重置一个已经存在的过程或者路由器软件重置 BGP 过程引起的。

▫任何状态中收到 Notification 报文或 TCP 拆链通知等 Error 事件后，BGP 都会转至 Idle 状态。

・在 Connect 状态下，BGP 启动连接重传定时器（Connect Retry），等待 TCP 完成连接。

▫如果 TCP 连接成功，那么 BGP 向对等体发送 Open 报文，并转至 OpenSent 状态。

▫如果 TCP 连接失败，那么 BGP 转至 Active 状态。

▫如果连接重传定时器超时，BGP 仍没有收到 BGP 对等体的响应，那么 BGP 继续尝试和其它 BGP 对等体进行 TCP 连接，停留在 Connect 状态。

・在 Active 状态下，BGP 总是在试图建立 TCP 连接。

▫如果 TCP 连接成功，那么 BGP 向对等体发送 Open 报文，关闭连接重传定时器，并转至 OpenSent 状态。

▫如果 TCP 连接失败，那么 BGP 停留在 Active 状态。

▫如果连接重传定时器超时，BGP 仍没有收到 BGP 对等体的响应，那么 BGP 转至 Connect 状态。

# BGP 注入路由的方式：

​ 两种：network 和 import-route

​ network：只要是本地 IP 路由表中存在的路由都可以宣告到 BGP 中（掩码要一致）

​ Import-route：可以将不属于 BGP 的路由引入，如直连，静态，OSPF 等

​ BGP 路由表中：* 表示有效路由 ，> 表示最优路由

​ BGP 发布路由时通过 Update 报文携带 NLRI（网络层可达信息，携带发布的路由前缀和前缀长度）和路径属性（用于选路和防环），撤销路由通过 Update 报文中的 withdraw，携带撤销的路由前缀和前缀长度

注：在 BGP 中，认为目的地与下一跳一致的路由无效

# BGP 的通告原则：
1. 只发布有效并且最优的路由给对等体
2. 从EBGP对等体学习到的路由可以传给所有邻居
3. 从IBGP对等体学习的路由不能传给其他的IBGP（IBGP水平分割，为了防环）
4. BGP和IGP同步（IBGP和IGP同步，当设备将BGP路由发布给IBGP对等体之前，会先检查本地IGP路由表是否存在该路由，如果存在则发送，不存在则不发，为了避免路由黑洞，华为默认永不开启）
# 路由黑洞：

由于 BGP 可以跨跳建立对等体关系，所以导致中间设备没有运行 BGP，数据到了这里之后由于没有路由被丢弃

# 解决方法：
1. IBGP 全互联
2. 将BGP引入IGP
3. 物理直连
4. 隧道
5. MPLS
