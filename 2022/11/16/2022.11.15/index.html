# OSPF 外部路由：

​ OSPF 外部路由通过 5 类 LSA 进行传递

​ 泛洪区域：整个 OSPF 区域

# 5 类 LSA（AS-external-lsa）：

  1. LS ID: 外部目的的网络地址

  2. Advertise：ABSR 的 router id

  3. Network mask：目的网络掩码

    ​ E：标识外部路由的类型

    ​ 0：外部路由为 1 类

    ​ 1：外部路由为 2 类

    ​ 1 类外部路由认为内部和外部的开销参考值一致，同时计算内部和外部的路由开销

    ​ 2 类外部路由认为内部和外部的开销参考值不一致，不计算外部开销，只计算内部开销

    ​ 默认情况下，外部路由使用 2 类

  4. metric：ASBR 去往目的网络的开销

  5. Forwarding address ：转发地址，用于解决次优路径，一般默认为 0.0.0.0

  6. external route tag ：给外部路由打上标签（tag），用于匹配做策略

# 4 类 LSA (ASBR-summary LSA):

​ 作用：告知其他区域 ASBR 的位置信息，以及怎么去往 ASBR

​ 泛洪区域：与 ABR 相邻的区域

​ LS ID：ASBR 的 route id，标识哪台 ASBR

​ Advertise ：ABR 的 router id ，

​ metric：开销，ABR 去往 ASBR 的开销

​

# OSPF 路由的优先级：

​ 域内路由 > 域间路由 > 1 类外部路由 > 2 类外部路由

# 外部路由计算：

​ 1 类外部路由：

​ 2 类外部路由：

  1. 查看对应 4 类 LSA 得到 ABR 去往 ASBR 的域间开销
  2. 使用 1 类和 2 类得到自身去往 ABR 的域内开销
  3. 将域间开销 + 域内开销 = 自身去往外部目的网段的开销
# OSPF 同步 LSDB 过程：
  1. 查看序列号，将序列号大的放入 LSDB 中进行路由计算，一致则查看第二步
  2. 查看 LSA 中的校验和，将校验和大的放入 LSDB 中进行路由计算，一致则进行第三步
  3. 查看生存时间，先查看是否存在生存时间为 3600s，存在则直接将 3600 的加入 LSDB 进行路由计算
  4. 如果没有 3600 则查看两条 LSA 的生存时间差值大于 900s
    1. 大于 900s，则将生存时间小的放入 LSDB 中，进行路由计算
    2. 小于 900s，则认为两条是一致的 LSA，不进行处理 l
# OSPF 的路由更新过程：

  1. 域内路由更新：

    路由器感知到拓扑发生改变，产生一条新的一类 LSA，序列号 + 1，生存时间重置

    网络类型为 MA 网络，则 DR 也会产生一条 2 类的 LSA，序列号 + 1，生存时间重置

  2. 域间路由更新：

    ABR 路由器感知到路由发生变化，产生一条新的 3 类 LSA，序列号 + 1，生存时间重置

  3. 域外路由更新：

    ASBR 路由器感知到路由发生变化，产生一条新的 5 类 LSA，序列号 + 1，生存时间重置

# OSPF 的路由撤销过程：

  1. 域内路由撤销：

    相关的路由器产生一条新的 1 类 LSA，序列号 + 1，LSA 中不再包含被撤销网络的 link 信息

    在 MA 网络中，则 DR 会产生一条 2 类 LSA，序列号不变，老化时间为 3600s

  2. 域间路由撤销：

    ABR 产生一条生存时间为 3600s 的 3 类 LSA，序列号不变，用于删除区域中其他路由的 LSDB 中自己原来产生的 3 类 LSA。

  3. 域外路由撤销：

    ASBR 产生一条生存时间为 3600s 的 5 类 LSA，序列号不变，用于删除区域中其他路由的 LSDB 中自己原来产生的 5 类 LSA。

    ASBR 还引入了其他的外部路由，则不会将对应的 4 类进行删除

    ASBR 没有引入外部的路由，ABR 删除对应的 4 类 LSA。

# 5 类 LSA 中的转发地址：

  1. 一般情况下，默认为 0.0.0.0

  2. 置位条件

    ASBR 连接外部路由的接口需要宣告到 OSPF 区域

    OSPF 域中有其他设备接口也处于同一网段

    FA 地址为 ASBR 去往外部网段的下一跳地址

# 特殊区域：

​ 为了减少设备的 LSDB 的 LSA 的数量

# stub 区域：

​ 规避 4 类和 5 类 LSA，通过一条缺省 3LSA 实现对外部路由的访问

​ 配置：在区域中的所有路由器上的区域视图下配置 stub 命令

​ 特点：不会存在 4 类和 5 类 LSA

​ 存在的 LSA: 1 类，2 类，3 类，3 类缺省

# Totally stub:

​ 规避 4 类，5 类和 3 类明细，通过 3 类缺省实现对外部和区域间路由的访问

​ 配置：在区域中的所有路由器上的区域视图下配置 stub 命令，在 ABR 上配置是添加 no-summuy 命令

​ 特点：不会存在 3 类明细，4 类，5 类 LSA

​ 存在的 LSA:1 类，2 类，3 类缺省

# nssa 区域：

​ 有需求引入外部路由信息，但是由于特殊区域规避了 4 类和 5 类 LSA

​ 通过添加新的 LSA 实现引入外部路由的需求

​ 配置：在区域中的所有路由器上的区域视图下配置 nssa 命令

​ 特点：不存在 4 类，5 类 LSA，同时实现了外部引入操作

​ 存在 LSA：1 类，2 类和 3 类明细，7 类，7 类缺省

​

# Totally nssa 区域：

​ 规避 4 类，5 类和 3 类明细，通过 3 类缺省实现对外部和区域间路由的访问，可以引入外部路由

​ 配置：在区域中的所有路由器上的区域视图下配置 nssa 命令，在 ABR 上配置是添加 no-summuy 命令

​ 特点：不存在 3 类明细，4 类，5 类 LSA，同时实现了外部引入操作

​ 存在 LSA：1 类，2 类，3 类缺省，7 类，7 类缺省

​

# 7 类 LSA（nssa LSA）:

​ 用于特殊区域引入外部路由

  1. LS ID: 外部目的的网络地址

  2. Advertise：ABSR 的 router id

  3. Network mask：目的网络掩码

  4. metric：ASBR 去往目的网络的开销

  5. Forwarding address ：转发地址，用于解决次优路径，一定为非零

    ​ 为 ASBR 连接 OSPF 区域内的接口 IP 地址

  6. external route tag ：给外部路由打上标签（tag），用于匹配做策略

# 7 类 LSA 转 5 类 LSA:

​ 执行路由器：连接 nssa 区域的 ABR 设备，如果有多个，则使用 router-id 最大的 ABR 设备

​ 当 ABR 执行 7 转 5 后，设备将变成 ASBR 设备

​ 7 类中的 forwarding address 用于解决次优路径

​ 7 转 5 后会继承 7 类中的 forwarding address，此时，其他区域中的路由器访问 nssa 区域引入的外部路由时，直接通过 forwarding address 找到最优路径。

​
