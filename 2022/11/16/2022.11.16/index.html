# 各类 LSA 的 LS ID, 产生者，泛洪区域等：

​

LSA 名称 LS ID 产生者 泛洪区域
1 类 LSA (router LSA) 产生者的 router id 所有运行了 OSFP 协议的路由器 路由器所处的区域内
2 类 LSA (network LSA) DR 的接口 IP 地址 DR 设备 路由器所处的区域内
3 类 LSA (network summary) 目的网段信息 ABR 设备 与 ABR 相邻的设备
4 类 LSA（ABSR summary LSA） ASBR 的 router-id ABR 设备 与 ABR 相邻的设备
5 类 LSA (AS-external LSA) 目的网段信息 ASBR 设备 除特殊区域外的整个 OSPF 域
7 类 LSA (nssa LSA) 目的网段信息 nssa 区域中的 ASBR 产生此条 LSA 的 nssa 区域
# OSPF 路由汇总：

作用：减少 OSPF 域中路由器上的 LSDB 中的 LSA 数量（3 类或 5 类）

  1. 域间路由汇总

    对区域传出的 3 类 LSA 进行汇总处理，必须在需要汇总区域路由的 ABR 上配置

    配置完成后，所有明细路由失效，不再向外传递明细的三类 LSA

    配置命令：[huawei-ospf-1-area-0.0.0.1] abr-summary 172.16.0.0 255.255.248.0 //ABR 上的区域视图下配置

  2. 域外路由汇总

    对于引入外部路由的 5 类进行汇总处理，必须在做了引入操作的 ASBR 上进行配置

    配置完成后，所有的明细路由失效，不再将明细的 5 类 LSA 进行传递

    配置命令：[huawei-ospf-1] asbr-summary 172.16.0.0 255.255.248.0 //ASBR 上的进程视图下配置

# 协议特征：

​ 静默接口：设置为静默接口后，不再发送和接收任何关于 OSPF 协议的报文

​ 不能从该接口建立邻居关系

​ 配置：在路由器上的协议视图下配置

​ [huawei-ospf 1] silent-interface g0/0/0

# ISIS:

​ ISIS（中间系统到中间系统）

​ 在 OSI 参考模型中，路由器称之为系统

​ 在 TCP/IP 参考模型中，路由器称之为节点

​ ISIS 工作在 OSI 七层模型中的网络层，TCP/IP 的数据链路层

​

​ NSAP: 网络服务访问点（类似于 IP 地址的作用）由 IDP 和 DSP 组成

​ IDP: 类似于 ip 地址中的网络地址

​ DSP: 类似于 IP 地址中的主机位，由三部分组成

​ High order DSP：用于标识区域

​ System ID：用于标识主机

​ SEL：标识服务类型，诺服务于 IP，则固定为 00

​ NET（网络实体名称）

​ 最短为 8 字节，最长为 20 字节

​ Area id ：1-13 字节，用于标识设备所处的区域

​ System id：系统 id，用于标识区域中的某台设备

​ 华为一般采用 router id 计算得出

​ 将 router id 的每一个位置填充为 3 位，、

​ 在重新划分每四个数位一组，得出 System id，

​ 项目中一般采用自行规划，不使用华为的方法

​ SEL：标识服务类型，服务 IP 固定为 00

​ 拿到 NET 地址后，建议从后往前看，将 SEL 和 System id 区分后，剩余部分为 Area id

​ 注：一台路由器最多可以配置三个网络实体名称

# ISIS 的路由器分类：
# Level 1 路由器：

​ 是一种区域内部路由器

​ 只能和同一区域的 level 1 和 level 1/2 路由器建立邻接关系

​ 只维护 level 1 的 LSDB

​ 访问其他区域时，通过 level 1/2 路由器进行访问

# Level 2 路由器：

​ 是一种骨干路由器

​ 可以和同一区域或者不同区域的 level 2 或 level 1/2 路由器 形成邻接关系

​ 维护者 level 2 的 LSDB

​ 转发源和目的地址都不属于骨干区域的流量，也就是穿越流量

​

# level 1/2 路由器：

​ 类似于 OSPF 中的 ABR 设备，用于连接骨干区域和非骨干区域

​ 可以和同一区域的 level 1 路由器或者 level1/2 形成 level 1 的邻接关系

​ 也可以和同一区域或者不同区域的 level 2 路由器或者 level 1/2 路由器形成邻接关系

​ 维护者 level 1 和 level 2 两个区域的 LSDB

​

# 支持网络类型：

​ 根据数据链路层协议的不同，所属的网络类型不同

# 广播：

​ 需要选举 DIS

# p2p (点到点)：

​ 不需要选举 DIS

# ISIS 的开销模式：

​ 根据不同的度量方式，开销取值范围不同

​ 窄度量：0-63

​ 宽度量：0-16777215

​ 更改开销的方式：

	1. 接口下直接更改
    		2. 设置全局开销
        		3. 自动计算开销：根据接口的带宽进行计算开销

​

# ISIS 报文格式：

​ 通用头部和专属头部

# 通用头部格式：（不用背）
  • Intradomain Routing Protocol Discriminator : 域内路由选择协议鉴别符，固定值设置为 0x83
  • Length Indicator ：整个 ISIS 的 PDU 长度
  • Version/Protocol ID Extension：版本 / 协议标识扩展，设置为 1（0x01）。
  • ID Length：NET 中的 System ID 区域的长度
  • R 位（Reserved）：保留，为 0
  • PDU type：标识具体是哪一种报文
类型值（需要背） 简称
15 L1 LAN IIH
16 L2 LAN IIH
17 P2P IIH
18 L1 LSP
20 L2 LSP
24 L1 CSNP
25 L2 CSNP
26 L1 PSNP
27 L2 PSNP

  • Version：版本，默认为 1

  • Maximum Area Address：最大支持区域个数，0-254

  • PDU specific header：专用头部，根据不同的专用头部，值不同

  • Variable length fields：携带不同的 TLV 实现不同的功能

# ISIS 具体报文：
**hello报文**：用于发现，建立，维护邻接关系

​ L1 hello：用于广播型网络中建立 level 1 邻接关系使用的 hello

​ L2 hello：用于广播型网络中建立 level 2 邻接关系使用的 hello

​ P2P hello：用于 P2P 网络中建立邻接关系使用的 hello

​ LSP 报文：用于同步 LSDB，其内容为链路状态信息

​ L1 LSP: 用于 level 1 邻接关系之间同步 LSDB

​ L2 LSP: 用于 level 2 邻接关系之间同步 LSDB

​ CSNP 报文：用于周期发送，同步所有路由器的 LSDB，包含所有 LSP 的摘要信息

​ L1 CSNP: 用于 level 1 的 DIS 设备或者 P2P 网络中进行发送

​ L2 CSNP: 用于 level 2 的 DIS 设备或者 P2P 网络中进行发送

​ PSNP 报文：用于请求和确认自身缺省的 LSP，进行同步 LSDB

​ L1 PSNP: 用于 level 1 的路由器请求和确认自身缺失的 LSP

​ L2 PSNP: 用于 level 2 的路由器请求和确认自身缺失的 LSP
