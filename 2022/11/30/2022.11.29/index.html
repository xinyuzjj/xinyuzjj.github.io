# Rstp：

​ 快速生成树协议，相比 STP 加快了收敛速度以及做了相应的改进措施

# STP 的不足：

  1. STP 中的阻塞端口没有言明是作为根端口还是指定端口的备份，再进入转发状态时还需要重新选举

  2. 端口状态中 Distable，Blocking，Listening，都不学习 MAC 地址和转发用户流量，对于用户来说作用一致

  3. STP 的 MAX Age 为 20s

  4. STP 非根交换机再转发配置 BPDU 时需要收到上游转发的 BPDU 才会继续向下转发

  5. STP 只有指定端口会处理次优 BPDU

  6. STP 中连接终端用户的端口也需要参与生成树计算，用户访问网络需要等待至少 30s 的时间

  7. STP 初始化收敛需要两个 Forwarding Delay 转发延迟时间 30s 才能进入稳定运行状态

  8. STP 认为一个端口 UP/Down 接收发生拓扑变化，由于发生变化的设备通知根桥后再由根桥通知全网

# RSTP 的改进：

  1. 将阻塞端口分为两种角色：AP 替代端口和 BP 备份端口，其中 AP 作为根端口的备份，BP 作为指定端口的备份

  2. 将端口状态前三种结合为一个 Discarding。Leraning 和 Forwarding 不变

  3. RSTP MAX Age 为三倍的 HelloTime 时间（6s）

  4. RSTP 中配置 BPDU 的转发完全自主进行（不需要等待上游）

  5. RSTP 只要收到了次优 BPDU 就会丢弃并回应自身缓存的最优 BPDU

  6. RSTP 新增了一个边缘端口角色，连接终端用户的端口设置为边缘端口后可以立即进入 Forwarding 转发状态，不需要参与生成树计算，但是如果边缘端口收到了 BPDU 就会丧失边缘端口属性，就会重新参与生成树计算

  7. P/A 机制：

  8. 设备之间互相交互 BPDU 后，

  9. 更优的一端会继续发送 P 置位的 RST BPDU，

  10. 下游收到 P 置位的 BPDU 后进入同步状态（除了接收到的端口和边缘端口外其他所有端口阻塞），

  11. 接收到 P 置位 BPDU 的端口进入转发状态，回复 A 置位 RST BPDU

  12. 上游收到 A 置位 RST BPDU 后端口进入转发状态；

    注：如果是收到 P 置位 RST BPDU 的端口是 AP/BP 则不会回复 A 置位 BPDU，上游由于无法收到 A 置位 RST BPDU 进入慢收敛状态，等待两个转发延迟时间 30s 进入转发状态。

  13. RSTP 认为一个非边缘端口进入 Forwarding 状态就是发生了拓扑变化，并且由发生变化的设备通知全网：

    1. 发生变化的端口向外发送 TC 置位 RST BPDU，启动一个计时器（2 倍 HelloTime 时间），超时后停止发送
    2. 收到 TC 置位 RST BPDU 的设备将除了接收到的端口和边缘端口外的其他所有端口学习到的 MAC 地址清空
    3. 继续向下游转发 TC 置位 RST BPDU
# RSTP 的保护功能：
1. BPDU保护：由于边缘端口收到BPDU后会丧失边缘端口属性，重新参与生成树计算，引起网络重新收敛，为了防止不法分子向边缘渡口发送BPDU，可以开启BPDU保护，开启保护功能后，边缘端口收到BPDU会立即关闭端口
2. 根保护：为了防止非法分子伪造更优的BPDU影响根桥的地位。可以再指定端口开启根保护，收到更优BPDU后会将端口状态置为Discarding，一段时间后没有再收到就可以回到Forwarding，如果一直收到则一直处于Discarding状态
3. 环路保护：为了防止根桥—>RP之间单向链路故障时造成的环路，开启环路保护后，根端口/AP无法收到根桥的BPDU之后会进入Discarding状态，直到重新收到BPDU
4. 防TC-BPDU攻击：由于RSTP设备在收到TC置位 RST BPDU 后删除MAC地址表项，非法分子利用这一特性向网络中发送大量TC RST BPDU，导致设备一直删除表项，数据都是未知帧形成泛洪，开启防TC-BPDU攻击后，可以设置设备单位时间内处理TC报文的次数，降低影响（例如非法分子1s发了10次，设置2S只处理1次）

1.stp 不会指定 ap 为 dp 或者 rp 的备份端口

2.stp 的 max age 为 20

3.stp 的三个端口状态 不会
