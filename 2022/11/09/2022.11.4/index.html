# AAA:

​ 认证，授权，计费的简称，是一种网络安全的管理架构，提供认证，授权和计费的功能

​ 认证：Authentication，对访问者的身份进行认证，确保访问者的身份合法性

​ 授权：Authorization，对不同用户进行不同的授权，限制其能够使用的网络服务或者说网络资源

​ 计费：Accounting，对用户使用网络服务的行为进行记录

# AAA 架构：C/S，C 是客户端 Client，S 是服务器 Server

​ Client：需要进行认证的设备

​ NAS：网络接入服务器，负载收集网络中用户的身份信息，提交给 AAA 服务器，并根据 AAA 服务器返回的结果进行相应的处理

​ AAA 服务器：存储用户信息，根据收到的用户信息查找数据库，然后返回给 NAS 认证结果，以及一些授权信息

# 认证方式：

​ 不认证：完全信任访问者的身份，不对用户身份进行认证

​ 本地认证：将用户信息存储在 NAS 上，此时 NAS 也作为 AAA 服务器，只需要在本地进行认证

​ 优点：

​ 部署成本低

​ 处理速度快

​ 缺点：

​ 不方便进行统一的配置和管理

​ 存储的用户信息受网络设备硬件限制

​ 远端认证：将用户信息存储在 AAA 服务器上，此时 NAS 会作为客户端将用户信息发送给 AAA 服务器进行认证，AAA 服务器再返回对应的认证结果

授权方式：

​ 不授权：▫不对用户进行授权处理。

​ 本地授权：▫根据 NAS 上对应域下的配置进行授权。

​ 远端授权：

计费方式：

​ 不计费：提供免费的网络服务，不对用户进行计费

​ 远端计费：使用远端的 AAA 服务器进行计费

域：AAA 是基于域来对用户进行管理，每个用户都会属于一个域，在每个域下可以配置单独的认证，授权和计费方案（模板）

​ 根据用户输入的用户名决定用户所属的域，用户名 @域名，如果没有后面的 @则表示是默认域

​ 普通用户：default

​ 管理用户：default_admin

AAA 使用的一些协议：

​ RADIUS: 公有

​ HWTACACS: 私有

​ LDAP:

Telnet 命令：
[r1]telnet server enable
[r2]user-interface vty 0 4
[r2-ui-vty0-4]authentication-mode aaa
[r1]aaa
[r2-aaa]local-user user02 password cipher huawei123 privilege level 3
[r2-aaa]local-user user02 service-type telnet

FTP 命令：
[r1]ftp server enable
[r1]aaa
[r1-aaa] local-user user01 password cipher huawei123
[r1-aaa] local-user user01 privilege level 3
[r1-aaa] local-user user01 ftp-directory flash:/
[r1-aaa] local-user user01 service-type ftp

Console 口的认证
[r2] user-interface console 0
[r2-ui-console0]authentication-mode aaa
[r2]aaa
[r2-aaa]local-user user02 service-type terminal
