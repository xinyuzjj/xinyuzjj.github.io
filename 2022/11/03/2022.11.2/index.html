# ACL:

​ 访问控制列表

​ 作用：通过对流量进行识别和分类，对不同类别的流量进行相对应的处理，提高网络安全性和服务质量

​ 本质：流量过滤

​ 服务质量的性能指标：

​ 带宽：带宽是指端到端通信链路上单位时间内传输的最大数据量

​ 时延：数据包从源端到目的端所需要的时间

​ 抖动：时延的变化就是抖动，时衡量网络稳定性的指标，越小代表网络越稳定

​ 丢包率：在数据传输过程中，丢弃报文占全部报文的百分比

​ 可用性：可靠性，HA

# ACL 的组成：

​ 规则编号：一个 ACL 下可以创建多个规则 RULE，使用规则编号进行标识

​ 手工指定：管理员手工指定规则编号

​ 自动生成：如果管理员没有指定的话，系统会根据步长自动进行匹配，步长默认 5

​ 处理行为：执行的动作

​ permit：允许通过

​ deny：拒绝通过

​ 匹配条件：匹配上该规则的条件，根据不同类型的 ACL 能够使用不同的条件

​ 比如源 / 目 IP, 源 / 目 MAC 地址，协议类型，时间，分片信息

# ACL 的分类：

​ 接口 ACL:1000-1999

​ 基本 ACL:2000-2999, 能够匹配报文的源 IP 地址，分片信息，时间

​ 高级 ACL:3000-3999, 能够匹配源 / 目 IP 地址，源 / 目端口，协议类型，时间

​ 五元组：源 IP 地址，目的 IP 地址，源端口，目的端口，协议类型（三层协议）

​ 二层 ACL：4000-4999，能够匹配源 MAC，目的 MAC，二层协议

​ 用户自定义 ACL:5000-5999

​ 用户 ACL:6000-6999

​ ACL 的匹配规则：

​ 按照规则编号从小到大匹配

​ 如果匹配上该规则的匹配条件，则执行该规则的相应动作，不再继续匹配

​ 如果没有匹配上该规则的匹配条件，则进行下一跳规则的匹配

​ 如果所有规则都没有匹配上，那么执行底层的默认动作

​ 如果是匹配数据流量，底层默认动作是允许所有

​ 如果是匹配路由信息，底层默认动作是拒绝所有

​ 通配符：

​ 如果匹配条件是 IP 地址的话，需要在后面配置对应的通配符，表示该规则匹配的 IP 地址范围，通配符 32bit 和 IP 地址类似，不要求是连续的 0 或者连续的 1 组成

​ 如果通配符取值为 0 的话，表示该比特位对应的 IP 地址比特位需要精确匹配

​ 如果通配符取值为 1 的话，表示该比特位对应的 IP 地址比特位任意匹配

​ 举例：

​ 192.168.1.1 0.0.0.0 ：匹配 192.168.1.1

​ 192.168.1.0 0.0.0.255 匹配 192.168.1.0-192.168.1.255

​ 192.168.1.0.0.0.254 匹配 192.168.1.0-192.168.1.255 的偶数地址

​ 192.168.1.1.0.0.254 匹配 192.168.1.0-192.168.1.255 的奇数地址

​ 0.0.0.0 255.255.255.255 ：所有地址

​

​ Traffic-Filter：流量过滤，属于一种调用工具，能够调用 ACL，本身并不具备动作，具体的动作由调用的 ACL 决定

​ ACL 配置：

​ 1. 创建 ACL acl number 3000

​ 2. 创建规则 rule 3 deny icmp source 192.168.1.3 0 destination192.168.2.2 0 time-range haha

​ 3. 接口使用 Traffic-Filter

​ [R1-G0/0/1] Traffic-Filter inbound acl 3000
