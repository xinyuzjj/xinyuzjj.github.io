# OSPF 报文格式：

​ OSPF 通用报文头部：

​ Version：版本

​ type：标识报文类型

Type报文名称报文功能
1Hello发现和维护邻居关系
2Database Description交互链路状态数据库摘要
3Link State Request请求特定的链路状态信息
4Link State Update发送详细的链路状态信息
5Link State Ack确认 LSA

​ Packet length：报文长度

​ router id：产生报文路由器的 router id

​ Area id ：产生报文的区域

​ CheckSum：校验和，用于检验数据包的完整性

​ Auth type：认证类型

​ Authentication：认证数据

# hello 报文：
字段长 度含义
Network Mask32 比特发送 Hello 报文的接口所在网络的掩码。
HelloInterval16 比特发送 Hello 报文的时间间隔。
Options8 比特可选项： E：允许 Flood AS-External-LSAs MC：转发 IP 组播报文 N/P：处理 Type-7 LSAs DC：处理按需链路
Rtr Pri8 比特DR 优先级。默认为 1。如果设置为 0，则路由器不能参与 DR 或 BDR 的选举。
RouterDeadInterval32 比特失效时间。如果在此时间内未收到邻居发来的 Hello 报文，则认为邻居失效。
Designated Router32 比特DR 的接口地址。
Backup Designated Router32 比特BDR 的接口地址。
Neighbor32 比特邻居，以 Router ID 标识。

​ Network Mask：发送 hello 报文的接口网段掩码

​ Hello Interval：接口发送 hello 报文的时间间隔，广播网络默认 10s

​ Options：可选能力参数

​ E: 支持传递外部路由

​ N/P: 支持 7 转 5 类 LSA

​ MC: 支持组播能力

​ DC: 支持按需链路

​ Router priority：接口 DR 优先级，如果优先级为 0，不参与选举 DR

​ Router Dead Interval：邻居失效时间，时间为 hello 间隔的 4 倍

​ Designated Router ：接口所处网段的 DR 的 ip 地址

​ Backup Designated Router：接口所处网段的 BDR 的 ip 地址

​ Active neighbor ：接口下的活跃邻居列表

​

DD 报文：

字段长度含义
Interface MTU16 比特在不分片的情况下，此接口最大可发出的 IP 报文长度。
Options8 比特可选项： E：允许 Flood AS-External-LSAs； MC：转发 IP 组播报文； N/P：处理 Type-7 LSAs； DC：处理按需链路。
I1 比特当发送连续多个 DD 报文时，如果这是第一个 DD 报文，则置为 1，否则置为 0。
M (More)1 比特当发送连续多个 DD 报文时，如果这是最后一个 DD 报文，则置为 0。否则置为 1，表示后面还有其他的 DD 报文。
M/S (Master/Slave)1 比特当两台 OSPF 路由器交换 DD 报文时，首先需要确定双方的主从关系，Router ID 大的一方会成为 Master。当值为 1 时表示发送方为 Master。
DD sequence number32 比特DD 报文序列号。主从双方利用序列号来保证 DD 报文传输的可靠性和完整性。
LSA Headers可变该 DD 报文中所包含的 LSA 的头部信息。

​ Interface MTU: 接口最大发送数据长度，两端不一致无法建立邻接关系

​ Options：

​ E: 支持传递外部路由

​ N/P: 支持 7 转 5 类 LSA

​ MC: 支持组播能力

​ DC: 支持按需链路

​ I：当一次性发送多个 DD 报文时，标识报文是否为第一个 DD 报文，是就置位为 1

​ M：当一次性发送多个 DD 报文时，标识报文是否为最后一个 DD 报文，是就置位为 0

​ master/slave：标识设备为 master 还是 slave 角色 ，置位为 1 标识发送者为 master 设备

​ DD sep：DD 报文序列号，用于保证报文传递可靠性

​ AN LSA hearder：所有 LSA 的头部，用于描述 LSDB 摘要信息

# LSR 报文：

​

字段长度含义
LS type32 比特LSA 的类型号。
Link State ID32 比特根据 LSA 中的 LS Type 和 LSA description 在路由域中描述一个 LSA。
Advertising Router32 比特产生此 LSA 的路由器的 Router ID。

​ LS type：标识 LSA 的类型

​ LS ID: 和 LS type ，advertise 唯一标识一条 LSA

​ advertise：始发者路由器，产生路由器

# LSU 报文：

​ Number of LSAs：标识 LSU 具体携带 LSA 数量

​ LSAs：LSA 的明确信息

# LS ACK 报文：
字段长度含义
LSAs Headers可变通过 LSA 的头部信息确认收到该 LSA。

​ AN LSA header ：具体收到所有的 LSA 的头部信息

# OSPF 邻居建立不成功的原因：
1. 版本 不一致
2. area id不一致
3. 认证类型不一致
4. 掩码不一致（广播网络中）
5. ip地址不处于同一网段
6. router id 一致
7. 认证数据不一致
8. 邻居失效时间
9. hello 时间间隔不一致
10. 可选参数不一致
11. 网络类型不一致
12. 接口未宣告OSPF
# OSPF 邻接关系建立不成功的原因：
  1. MTU 值不一致
# OSPF 认证类型：

​ 区域认证：区域中的所有设备都需要配置区域认证，对区域中的 OSPF 报文进行认证

​ 认证类型和数据均在 OSPF 通用头部中

​ 接口认证：对接口下收发的 OSPF 报文进行认证，认证类型和数据均在 OSPF 通用头部中

# OSPF 认证方式：

​ 不认证：不对 OSPF 报文进行认证

​ 简单明文：对 OSPF 报文进行明文认证，使 OSPF 报文携带明文的密钥进行认证，安全性不高

​ MD5: 对密钥进行 MD5 计算后，放入报文中，进行认证，安全性更高

​ keychain：周期性更换密钥和加密算法，达到动态更新密钥的能力

# OSPF 网络类型：
OSPF 网络类型常见链路层协议是否 选举 DR是否和邻居建立邻接关系
P2P (点对点)PPP 链路 ；HDLC 链路否是
Broadcast以太网链路是DR 与 BDR 、DRother 建立邻接关系 BDR 与 DR 、 DRother 建立邻接关系 DRother 之间只建立邻居关系
NBMA （非广播多路访问）帧中继链路是DR 与 BDR 、DRother 建立邻接关系 BDR 与 DR 、DRother 建立邻接关系 DRother 之间只建立邻居关系
P2MP （点对多点）需手工指定否是
