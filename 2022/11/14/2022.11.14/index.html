# 链路状态信息的通用头部：

​ LS age：生存时间，最大 3600s，按时间增加，每到 1800s 时将由产生这条 LSA 的路由器重新产生

​ Option：可选项，一些可选参数

​ LS type：LSA 的类型，1，2，3，4，5，6，7

​ LS ID：每一种 LSA 的对应不同的含义

​ Advertise router：始发者路由器，用于描述 LSA 是谁产生的，router id

​ 注：LS type，LS ID ，advertise router 三元组，用于唯一标识一条 LSA

​ LS sequence number：序列号，用于比较 LSA 的新旧，取值 0x80000001-0x7FFFFFFF

​ LS checksum：校验和，用于检验 LSA 的完整性，并且用于比较 LSA 的新旧

​ Length：整个 LSA 长度

# 区域内路由计算：

​ 一类 LSA（router lsa）：运行了 OSPF 的每一台路由器都会产生一条，用于描述自己的拓扑和网络信息 作用：用于区域内的路由计算

​ LS ID ：产生者的 router id

​ V：当置位为 1 时，代表一类产生的类型为虚连接

​ E：当置位为 1 时，代表产生者是 ASBR 设备

​ B：当置位为 1 时，代表产生者时 ABR 设备

​ links：用于标识一类 LSA 携带 link 数量

​ link id: 不同网络类型代表不同的含义

​ link data: 不同网络类型代表不同的含义

Link type Link ID Link data
P2P 对端邻居的 router-id 自身接口 IP 地址
trasnet DR 的接口 ip 地址 自身接口 IP 地址
subnet 接口所属网段 接口 IP 的掩码
v-link 对端邻居的 router-id 自身接口 IP 地址

二类 LSA (network LSA): 由 DR 产生，用于描述 MA 网络的拓扑信息和路由信息，

​ 作用：用于计算区域内的路由信息

​ Link State ID ：DR 的接口 IP 地址。

​ Network Mask：MA 网络的子网掩码

​ Attached Router：形成邻接关系的设备的 router id

​

# 区域间路由计算：

​ 三类 LSA (network summary LSA) ：由 ABR 产生，用于传递不同区域之间的路由信息

​ 作用：用于区域间的路由计算 是

​ LS ID: 目的网段信息

​ advertise router ：ABR 的 router id ，

​ Network mask：目的网络掩码

​ metric：ABR 去往目的网络的开销

# 区域间路由计算方式：
1. 由ABR去往目的网段的3类LSA传递到区域中
2. 区域中的路由器收到3类LSA后，得到ABR去往目的网段的开销
3. 根据区域内的SPF计算得出，路由器自身去往ABR的开销
4. 将自身去往ABR的开销+ABR去往目的网段的开销=自身去往目的网段开销
