# ISIS 建立邻接关系的原则：
1. 只有同一层次的路由器之间才能建立邻接关系（level 1/2 即属于level 1 有属于level 2）
2. 对于level 1 路由器来说，area id 必须一致
3. 两端设备的ISIS网络类型必须一致
4. ISIS两端的ip地址需要处于同一网段（特殊情况除外- - ->p2p）
# hello 报文格式 ：

  • Reserved/Circuit Type：用于表示路由器类型

  • Source ID：发送设备的 System id

  • Holding Time ：邻接关系的失效时间，一般为 3 倍的 hello 间隔

  • PDU length：整个报文的长度

  • Priority：优先级，用于选举 DIS 时使用

  • LAN ID:DIS 的 system id 加上一个字节的 01/10, 只存在于广播网络中

  • Local Circuit ID：本地链路 ID，用于 P2P 网络类型协商

  • Variable Length Fields：可选择携带的某些参数值或者可选功能

    ​ 采用 TLV 结构进行携带，需要提供某种功能时，只需要加载对于对应的 TLV 即可

# 广播型网络邻接关系的建立：（三次握手）

​ R1<——————————>R2

1. 设备初始状态都处于down状态
2. 运行或激活接口ISIS后，发送hello报文，假设R1先发送Hello 报文
3. R2收到R1的hello 报文后，状态转变为initial ，同时将R1的System id 写入自己hello 的邻接列表
4. R2 发送hello 报文，邻接列表中含有R1的System id
5. R1收到后，将R2列为自己的邻接路由器，同时将R2的System id 列入自己的hello 报文中，发送给R2
6. R2收到R1 发送的hello 报文后，将自己的状态转变为UP状态，此时邻接关系建立完成
# 点到点网络（两次握手，默认采用三次握手）

​ R1<——————————>R2

  1. 设备初始状态都处于 down 状态
  2. 运行或激活接口 ISIS 后，发送 hello 报文，假设 R1 先发送 Hello 报文
  3. R1 发送 hello 报文后，报文中的邻接列表为空
  4. R2 收到 R1 的 hello 报文后，直接将 R1 列为自己的邻接路由器，并将状态转变为 UP
  5. R2 发送 hello 报文，报文中的邻接列表中有 R1 的 System id
  6. R1 收到 R2 的 hello 报文后，将 R2 列为自己的邻接路由器，邻接关系建立完成

配置接口使用两次握手建立邻接关系：

配置接口交互 hello 次数：isis ppp- negotiation 2-way

PPP 链路两端 IP 不处于同一网段使用

[huawei-serial 1/0/0] isis peer-ip-ignore // 配置 PPP 链路不检查对端 IP 地址

# DIS 和伪节点：
1. DIS通过所有路由器选举产生
2. 负责产生伪节点的LSP
3. 用于周期发送CSNP报文，同步网络中的路由器的LSDB
4. 伪节点LSP使用DIS的system id 和 circuit id 进行标识，当circuit id 为非零时，表示LSP为伪节点LSP
# DIS 的选举规则：

  1. 比较设备的优先级，以大为优，优先级越高越容易成为 DIS 设备

    设备优先级：0-127，默认 63，ISIS 的优先级为 0 的设备参与选举 DIS

  2. 优先级相同时，比较 MAC 地址，MAC 地址大的作为 DIS 设备

ISIS 的 hello 报文间隔时间为 10s，DIS 的 hello 时间间隔为 10/3, 邻接失效时间 3 倍 10s

# ISIS 的 DIS 和 OSPF 的 DR 的对比：
1. 优先级为0的设备参不参与选举
2. 抢占，DIS支持抢占，DR不支持
3. 选举规则不同，优先级相同时，DIS比较MAC地址,DR比较router id
4. hello 时间间隔，DR的时间间隔为10/3,死亡时间
5. 有无备份,DR存在备份BDR,DIS不存在备份
6. 作用不同，DR的作用-减少邻接关系，DIS的作用-周期发送CSNP同步LSDB
7. 选择过程不同，DR选举过程为其他路由器感知自己为DR设备，DIS初始化默认自己为DIS
# LSP 报文格式：900s
  • PDU length：报文的长度
  • Remaining Lifetime：生存时间，时间范围 0 —1200 , 时间为递减
  • LSP ID：由 System id，伪节点 id，LSP 分片 id 组成，用于标识一条 LSP
  • Sequence number ：序列号，与 OSPF 的 LSA 的序列号作用一致
  • check sum：校验和，用于检验 LSP 的完整性
  • P：是否支持自动修复
  • ATT：由 level 1/2 路由器产生的 LSP 置位，用于 level 1 区域的路由器访问 level 2 区域使用
  • OL(overload)：过载比特位，当 LSP 的 OL 置位后，其他路由器将不会使用 LSP 进行路由计算
  • IS type：标识生成此条 LSP 的路由角色
