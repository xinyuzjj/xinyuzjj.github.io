# 交换机堆叠与集群

堆叠：istack，可以将多台支持堆叠特性的物理交换机虚拟为一台逻辑交换机（一般是盒式交换机）

集群：CSS，可以将两台支持集群特性的物理交换机虚拟为一台逻辑交换机（一般是框式交换机）

# 堆叠系统：

通过堆叠技术形成的虚拟交换机由多台物理交换机组成，交换机角色分为主交换机，备交换机，从交换机

​ 主交换机：负载收集成员交换机信息以及管理成员交换机

​ 备交换机：会同步主交换机的信息，作为主交换机的备份

​ 从交换机：主要负载业务数据的转发

# 堆叠 ID：

在对端系统中唯一标识一台交换机，缺省堆叠 ID 为 0，当一个堆叠系统中堆叠 ID 冲突时，会由主交换机重新分配

# 堆叠连接方式：

​ 堆叠卡堆叠：通过专用的堆叠接口和堆叠线缆连接进行堆叠

​ 业务口堆叠：利用现有的业务接口连接，再将业务接口加入堆叠逻辑接口进行堆叠

# 堆叠逻辑接口：

stack port n/1 和 stack port n/2 ，其中 n 为设备的堆叠 ID，再进行堆叠时，设备连接其他物理设备的逻辑接口必须是 n/1 连接 n/2（n/2 连接 n/1）

# 堆叠拓扑类型：

​ 链形连接：容易增加新的成员设备，但是可靠性更低（某条链路 / 设备故障堆叠系统就断开了）

​ 环形连接：增加新的设备需要先断开环，然后增加，但是可靠性更高

# 堆叠系统组建过程：
1. 物理连接，上电
2. 选举主交换机
3. 主交换机收集拓扑信息，分配堆叠ID等
4. 将信息同步给成员交换机后稳定运行
# 主交换机选举：
  1. 比较堆叠系统启动时间，已经稳定运行的优先与处于启动状态的交换机，20s 以内认为是同时启动
  2. 比较堆叠优先级，越大越优
  3. 比较 MAC 地址，越小越优
# 堆叠成员退出：
  1. 从交换机退出，更新拓扑信息进行同步即可
  2. 备交换机退出，重新选举新的备交换机，然后更新拓扑信息
  3. 主交换机退出，备交换机成为新的主交换机，再选举一个备交换机，更新拓扑信息
# 堆叠成员加入：

​ 一个未上电的设备连接到堆叠系统中之后再启动，自动成为从交换机

# 堆叠合并：

​ 两个稳定运行的堆叠系统连接再一起，两个主交换机之间进行竞选，竞选成功的继续为主交换机，竞选失败的成为从交换机

# 堆叠分裂：

​ 一个堆叠系统断开后成为多个堆叠吸引

1. 主备交换机在一个堆叠系统的更新拓扑信息
2. 主备不在一个堆叠系统，主交换机所在可以重新选举备交换机，备交换机所在备交换机成为主交换机，重新选举新的备交换机
3. 分裂后只有从交换机的重新选举主交换机和备交换机

注：堆叠系统分裂会造成地址冲突的问题，导致网络故障

# MAD：

多主检测，检测堆叠分裂后的冲突问题，分裂后的堆叠系统之间进行竞选，竞选成功的正常运行，竞选失败的关闭所有物理端口（除了管理员手动保留的）

# 检测方式:

​ 直连检测：

  1. 中间设备直连检测，通过一台专用的物理设备进行 MAD 检测，不承载其他业务，所有成员交换机与中间设备相连

  2. Full-mesh 直连检测，所有成员交换机两两互连，通过检测链路交互 MAD 报文检测

    注：直连检测在分裂前不交互报文，分裂后每 1s 交互 MAD 检测报文

代理检测：

  1. 单机代理检测：连接方式与中间设备直连检测一直，检测设备检测同时可以承载业务

  2. 堆叠系统互为代理：不同堆叠系统之间互相作为 MAD 检测设备，交互 MAD 报文

    注：代理检测在发生分裂前每 30s 交互 MAD 检测报文，分裂后每 1s 交互

# MAD 竞选过程：
  1. 多个堆叠系统的主交换机比较启动时间，先启动的优先竞选成功，20s 内认为相同
  2. 主交换机之间比较堆叠优先级，大的竞选成功
  3. 优先级相同比较 MAC 地址，小的竞选成功
# 堆叠升级：
1. 智能升级：所有成员交换机同步主交换机的软件版本和配置信息
2. 传统升级：设置好堆叠系统下一次启动的软件版本和配置信息，然后进行重启，会造成业务中断
3. 平滑升级：将堆叠系统划分为Active区域和Backup区域，Backup区域升级时Active区域承载业务，backup区域升级完再升级Active区域，此时Backup区域承载业务，可以保证业务不中断
# 跨设备链路聚合：

​ 设备通过堆叠技术形成一个虚拟交换机后，不同物理交换机上的链路可以进行跨设备的链路聚合，可以对流量进行负载分担

# 流量本地优先转发：

​ 对于成员交换机在转发数据时会进行负载均衡，一部分数据会通过堆叠线缆转发给其他成员交换机承载，此时会造成堆叠线缆负担过重，利用设置流量本地优先转发，只要本交换机存在出接口能够转发数据就不会让其他成员交换机转发

# 集群：

CSS，将两台支持集群特性的框式交换机虚拟为一台逻辑交换机

# 结构：

主交换机，备交换机

# 集群 ID：

在集群中唯一标识，默认集群 ID 为 1

# CSS link：

集群链路，集群成员交换机之间作用集群链路

# 集群优先级：

默认为 1，越大越优选举为主交换机

# 集群建立方式：

​ 传统 CSS: 主控板集群卡建立集群和业务口集群，集群卡需要通过专用的集群线缆建立集群，业务空需要加入集群逻辑接口进行集群

​ CSS2: 第二代集群，通过交互网板集群卡集群

注：传统 CSS 在转发流量时，同框 / 不同框流量都需要经过主控板，所以两台成员交换机都必须至少有一个可用的主控板

同框数据经过本地接口版 — 本地主控版 — 本地接口

不同框数据经过本地接口版 — 本地主控板 — 本地集群卡 — 对端集群卡 — 对端主控板 — 对端接口板

注：CSS2 转发数据直接通过交换网板，不需要经过主控板，实现了转控分离，具有 “集群主控 1+N 备份” 特性（整个集群只要有一个主控板能正常工作集群就能正常运行）
